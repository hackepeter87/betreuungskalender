# Personal iCalendar feed

Betreuungskalender can create a protected, read-only iCalendar feed for the
signed-in user. Calendar clients can subscribe to the generated `.ics` URL.
This is not CalDAV and does not allow edits from calendar applications.

## Access model

The feed uses a revocable per-user token:

- The token is generated by the authenticated app user.
- The database stores only a SHA-256 hash of the token.
- Rotating the feed URL revokes previous active tokens for that user.
- Revoking the feed immediately invalidates the current URL.
- The token grants access only to `/calendar/<token>.ics`, not to `/api/*`.

Treat the feed URL like a password. Anyone who has the URL can read the feed
until it is revoked.

## Scope

The feed contains non-deleted, non-cancelled care entries whose `created_by`
actor is the feed owner. In the current data model this is the only stable
per-user ownership signal for care times.

The feed deliberately excludes notes, evidence references, trips, costs, audit
data, internal actor history, and deleted or cancelled entries. Event titles
use the compact form:

```text
Kinder bei <Anzeigename>
```

Events use stable UIDs derived from the care-entry ID and include start, end,
`DTSTAMP`, and `LAST-MODIFIED` values.

## Subscribe

1. Sign in to Betreuungskalender.
2. Open `Einstellungen`.
3. In `Persönlicher Kalenderfeed`, choose `Feed-URL erzeugen`.
4. Copy the URL shown directly after creation.
5. Add it as a subscribed calendar in the calendar application.

Common clients call this "calendar subscription", "subscribe by URL", or
"add calendar from URL". Do not import the file as a one-time static calendar
unless the client has no subscription mode.

## Rotate or revoke

Use `Neue URL erzeugen` when the URL may have been shared unintentionally.
Existing calendar subscriptions using the old URL will stop updating.

Use `Feed widerrufen` to disable the current feed completely. A revoked URL
returns `404`.

## Operations

The feed endpoint is rate-limited with `RATE_LIMIT_EXPORT_MAX` and
`RATE_LIMIT_WINDOW_MS`. Application request logging redacts
`/calendar/<token>.ics` paths before writing request metadata.
