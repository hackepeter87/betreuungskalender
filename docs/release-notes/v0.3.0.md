# v0.3.0 - SQLite persistence and legacy migration

This release prepares Betreuungskalender for consistent self-hosted operation
with SQLite and the Fastify API as the single source of truth for domain data.
It also provides a guarded migration path for users of older browser-only
versions.

> **Release status:** Prepared but not yet published. The repository version is
> `0.3.0`, but the release becomes public only after the corresponding Git tag
> and GitHub release are created.

## Highlights

- SQLite/API-backed persistence for children, care entries, holidays, contact
  patterns, trips, costs, unavailable periods, settings, monthly closings, and
  audit records
- Central API client and asynchronous application store
- Explicit loading, error, and server-connectivity states
- Blocked write actions when the backend is unavailable
- Guided migration assistant for the legacy
  `betreuungskalender:data:v1` browser key
- Import preview with conservative duplicate and conflict detection
- Transactional append and replace modes
- Required SQLite backup before replace mode
- Migration audit entries and exportable migration reports
- Automated migration, rollback, invalid-data, and backup-failure tests

## Legacy browser-data migration

The application checks for the known legacy `localStorage` key after the API is
available. Legacy data is read only as a migration source; it is never used as
current domain persistence and is not deleted automatically.

The migration assistant supports:

- preview only
- append while skipping potential duplicates by default
- replace after a successful SQLite backup
- ignore without deleting the legacy data

Potential duplicates and conflicts are shown before any data is written.
Imports run in a server-side SQLite transaction, so a failed migration does not
leave a partial data state.

Detailed operator guidance is available in
[docs/migration.md](../migration.md).

## Upgrade notes

1. Create and verify an SQLite backup.
2. Keep an additional JSON export when upgrading an existing installation.
3. Install exact dependencies with `npm ci`.
4. Run `npm run build` and `npm test`.
5. Start the application; database migrations run automatically.
6. If legacy browser data is detected, use **preview only** before importing.
7. Create a fresh JSON backup after a successful legacy migration.

## Security

- The transitive development dependency `shell-quote` is updated to `1.8.4`,
  resolving CVE-2026-9277.
- Migration audit metadata records counts and outcomes without storing complete
  notes, evidence text, or personal domain records.
- The release check continues to block tracked databases, backups, exports,
  reports, environment files, and other sensitive artifacts.

## Limitations

- The application remains a documentation aid, not an official record or legal
  service.
- Migration duplicate and conflict detection is conservative and requires user
  review.
- The frontend service worker does not make server-side writes available
  offline.
- Automated end-to-end coverage for desktop, iPhone, and iPad remains planned
  in issue #5.
