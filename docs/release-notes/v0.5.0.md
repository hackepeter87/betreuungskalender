# v0.5.0

## Highlights

v0.5.0 strengthens the supported self-hosted operating path. It adds a
Compose-first update workflow with verified backups, archive checksums,
readiness validation, and paired runtime/database rollback. The API now also
has centrally configured rate limits for normal reads, writes, imports,
migrations, and exports.

## Operations and updates

- Use the supplied Compose layout as the primary production installation path.
- The updater supports a non-writing `--dry-run` preflight, an update lock,
  archive-content validation, SHA-256 validation, and disk-space checks.
- Every managed update creates and verifies a SQLite backup before switching
  the runtime.
- If the new runtime does not become healthy and ready, the updater restores
  the matching prior runtime and SQLite backup. It never attempts reverse
  migrations.
- Release archives include the built frontend and backend, the release-runtime
  Dockerfile, Compose definition, update helpers, and operator documentation.

## Security

- Fastify enforces per-client API rate limits. Writes, imports, migrations, and
  exports have stricter limits than ordinary reads.
- Limits are configurable through documented positive environment variables.
  The default store is process-local; horizontally scaled deployments require a
  separately reviewed shared store.
- CodeQL rate-limit findings are resolved through enforced route policy, not
  query suppression or alert dismissal.
- Repository CI workflows use current Node 24-runtime GitHub Actions while the
  application and release checks continue to use Node.js 22.

## Upgrade from v0.4.0

1. Obtain the matching `betreuungskalender-v0.5.0.tar.gz` archive and
   `.sha256` sidecar from the release.
2. Verify the checksum before extraction or execution.
3. Run the updater with `--dry-run` first and review the release notes.
4. Run the managed update from the existing release with the exact archive and
   checksum paths.
5. Confirm `/api/health` and `/api/ready` after the update.

Existing SQLite databases and backups remain compatible. No application schema
migration is introduced by this release.

## Backup and rollback

Create and verify a current backup before updating. The automated path verifies
the backup and restores it together with the prior runtime if post-update
validation fails. Retain the update log, failed archive, and backup metadata
when recovery requires investigation.

## Compatibility and limitations

- Docker Compose is the supported automated update path.
- Direct Node.js/systemd deployments remain a documented manual fallback.
- The release does not publish a container image to GHCR.
- Imported external calendars remain read-only and do not affect care reports,
  statistics, exports, or month closure.
- This application documents user-entered care data and does not provide legal
  advice or an authoritative legal assessment.
