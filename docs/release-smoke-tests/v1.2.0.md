## Runtime smoke test

The archive was extracted into an isolated temporary directory and installed
with production dependencies:

```bash
npm ci --omit=dev
```

Result: passed with `0 vulnerabilities`.

The published runtime was started from the extracted archive with a temporary
SQLite database and backup directory:

```bash
NODE_ENV=production \
HOST=127.0.0.1 \
PORT=34120 \
DATABASE_PATH=/tmp/betreuungskalender-v1.2.0-runtime-data/app.sqlite \
BACKUP_DIR=/tmp/betreuungskalender-v1.2.0-runtime-backups \
LOG_LEVEL=warn \
npm run start
```

The health endpoint reported:

```json
{"status":"ok","version":"1.2.0","databaseReachable":true}
```

Runtime verification passed:

```bash
HEALTHCHECK_URL=http://127.0.0.1:34120/api/health \
DATABASE_PATH=/tmp/betreuungskalender-v1.2.0-runtime-data/app.sqlite \
BACKUP_DIR=/tmp/betreuungskalender-v1.2.0-runtime-backups \
npm run verify:runtime -- --expected-version 1.2.0
```

Result: `Runtime verification successful: version 1.2.0, migrations applied.`

Readiness reported `status=ready`, `databaseReachable=true`, and
`migrationsApplied=true`.
