# v1.4.3 - Demo testing data and mobile auth menu

## Summary

`v1.4.3` packages the demo/testing improvements that landed after `v1.4.2`.
It keeps the native OIDC security model unchanged while making the demo
environment easier to validate and the mobile authenticated header usable on
iPhone-sized screens.

## Changes

- Added `DEMO_DATASETS_ENABLED` as a disabled-by-default option for demo and
  staging environments.
- Added an admin-only settings action that loads a fully synthetic edge-case
  dataset covering month boundaries, recurring contact rules, cancelled entries,
  additional care, costs, trips, external calendar overlays, unavailability
  warnings, month closures, and audit display.
- Replaced the crowded mobile authenticated-session header card with a compact
  user icon and submenu that shows display name, role, and `Abmelden`.
- Kept desktop session display, native OIDC POST logout, and external logout
  links unchanged.

## Upgrade notes

- No database migration is introduced by this release.
- Keep `DEMO_DATASETS_ENABLED=false` or unset in production.
- Demo/staging may set `DEMO_DATASETS_ENABLED=true`; loading the edge-case
  dataset intentionally replaces the current domain data.
- `latest` remains reserved for production and is not part of testing
  promotion.

- `npm run test:e2e -- --project=desktop`
