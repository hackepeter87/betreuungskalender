# v0.5.0 published artifact smoke test

Date: 2026-06-28

Release: <https://github.com/hackepeter87/betreuungskalender/releases/tag/v0.5.0>

## Scope

This smoke test validated the published `v0.5.0` release assets outside the
normal CI workflow. It used only fictional data and isolated temporary files.

## Artifact verification

- Downloaded `betreuungskalender-v0.5.0.tar.gz` from the GitHub release.
- Downloaded `betreuungskalender-v0.5.0.tar.gz.sha256` from the GitHub release.
- Verified the archive checksum with `shasum -a 256 -c`.
- Confirmed the archive checksum:
  `968509fecfd453b3dda7969adeba0880edaf5345e2e0b5e597cf759373511898`.
- Confirmed `package.json` reports version `0.5.0`.
- Confirmed `docs/release-notes/v0.5.0.md` is present and identifies `v0.5.0`.

## Archive contents

The archive contained the expected runtime files:

- built frontend in `dist/`
- built backend in `dist-server/`
- `Dockerfile.release`
- `deploy/compose.yml`
- operational scripts including `scripts/update.js`, `scripts/backup.js`,
  `scripts/restore-check.js`, `scripts/healthcheck.js`, and
  `scripts/runtime-verify.js`
- release notes and operator documentation

No prohibited data artifacts were found. The archive did not contain SQLite
databases, backup directories, generated exports, `.env`, secrets, private keys,
PDF reports, CSV exports, or real ICS files. The only environment file was the
intended `.env.example`.

## Runtime smoke test

The archive was extracted into an isolated temporary directory and installed
with production dependencies:

```bash
npm ci --omit=dev
```

Result: passed with `0 vulnerabilities`.

The published runtime was started with a temporary SQLite database and backup
directory. The following checks passed:

```bash
npm run healthcheck
npm run verify:runtime -- --expected-version 0.5.0
```

The health endpoint reported version `0.5.0`, database connectivity was
available, and readiness confirmed that migrations were applied.

## Backup and restore smoke test

A fictional child record named `Testkind v0.5 Smoke` was created through the API.
The SQLite backup script then created a temporary backup:

```bash
npm run backup
npm run restore:check
```

Result: passed. The generated backup file used mode `0600`.

The backup was copied into a second isolated runtime data directory. A separate
runtime instance started from that restored database successfully passed
`npm run verify:runtime -- --expected-version 0.5.0`, and the fictional child
record was available through the API.

## Update dry-run

A synthetic Compose root was created with temporary `data/`, `backups/`,
`releases/`, `.env`, and `compose.yml` paths. Docker Compose was available
locally.

The updater was run in dry-run mode against the published archive and checksum:

```bash
node scripts/update.js \
  --root /tmp/betreuungskalender-v0.5.0-smoke/compose-root \
  --version 0.5.0 \
  --artifact /tmp/betreuungskalender-v0.5.0-smoke/betreuungskalender-v0.5.0.tar.gz \
  --checksum /tmp/betreuungskalender-v0.5.0-smoke/betreuungskalender-v0.5.0.tar.gz.sha256 \
  --dry-run
```

Result: passed. The updater reported `preflight_passed` and `dry_run_complete`.
No release switch, deployment, or production update was performed.

The published `v0.5.0` artifact passed the manual smoke test. No artifact defect
or patch-release blocker was found.
