# Screenshots

Repository screenshots must use obviously fictional data and must not contain
real children's names, dates of birth, addresses, schools, schedules, notes,
evidence references, authentication identities, or browser bookmarks.

No static screenshots are currently committed. This avoids publishing stale UI
or accidental personal data. Before adding one:

1. Start with the built-in fictional demo data.
2. Inspect every visible value and browser chrome.
3. Crop or redact unrelated information.
4. Store optimized images under `docs/assets/`.
5. Add descriptive alt text and update README.

Suitable future views are the dashboard, mobile agenda, care-entry form,
unavailable-period form, and neutral report preview.
