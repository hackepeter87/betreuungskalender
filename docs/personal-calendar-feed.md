# Personal iCalendar feed

Betreuungskalender can create protected, read-only iCalendar feeds for the
signed-in user. Calendar clients can subscribe to the generated `.ics` URLs.
This is not CalDAV and does not allow edits from calendar applications.

## Access model

The feed uses a revocable per-user token:

- The token is generated by the authenticated app user for one selected feed
  scope.
- The database stores only a SHA-256 hash of the token.
- Rotating the feed URL revokes previous active tokens for the same user and
  scope.
- Revoking the feed immediately invalidates the current URL for the selected
  scope.
- The token grants access only to `/calendar/<token>.ics`, not to `/api/*`.

Treat the feed URL like a password. Anyone who has the URL can read the feed
until it is revoked.

## Scopes

New tokens can use these scopes:

- `all`: all active, non-cancelled care entries visible to the feed owner.
  In solo mode this is the whole calendar. In shared mode, non-admin users are
  limited to their assigned care parties.
- `party:<id>`: active, non-cancelled care entries assigned to one selected
  care party.
- `legacy`: existing pre-v1.5 tokens. These remain valid and keep their old
  `created_by` owner behavior until rotated or revoked.

The feed deliberately excludes notes, evidence references, trips, costs, audit
data, internal actor history, and deleted or cancelled entries. Event titles
use a compact form:

```text
Kinder bei <Anzeigename>
```

For party-scoped feeds, `<Anzeigename>` is the care-party name. For the
all-scope feed, the calendar name identifies the overall Betreuungskalender
feed.

Events use stable UIDs derived from the care-entry ID and include start, end,
`DTSTAMP`, and `LAST-MODIFIED` values.

## Subscribe

1. Sign in to Betreuungskalender.
2. Open `Einstellungen`.
3. In `Persönlicher Kalenderfeed`, choose the feed scope.
4. Choose `Feed-URL erzeugen`.
5. Copy the URL shown directly after creation.
6. Add it as a subscribed calendar in the calendar application.

Common clients call this "calendar subscription", "subscribe by URL", or
"add calendar from URL". Do not import the file as a one-time static calendar
unless the client has no subscription mode.

## Rotate or revoke

Use `Neue URL erzeugen` when the URL may have been shared unintentionally.
Existing calendar subscriptions using the old URL for that scope will stop
updating.

Use `Feed widerrufen` to disable the current selected feed. A revoked URL
returns `404`.

## Operations

The feed endpoint is rate-limited with `RATE_LIMIT_EXPORT_MAX` and
`RATE_LIMIT_WINDOW_MS`. Application request logging redacts
`/calendar/<token>.ics` paths before writing request metadata.
