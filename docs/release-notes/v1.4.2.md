# v1.4.2 - Native OIDC provider logout

## Summary

`v1.4.2` fixes native OIDC logout for Keycloak-backed deployments. Logout now
revokes the local Betreuungskalender session, clears the app cookie, and then
navigates the browser to the provider end-session endpoint so the Keycloak SSO
session is also ended.

Without this patch, refreshing the protected app after logout could silently
sign the user in again because Keycloak still had an active browser session.

## Upgrade notes

- Add `OIDC_POST_LOGOUT_REDIRECT_URI=https://<app-host>/` to native OIDC
  deployments, or rely on the default derived from `OIDC_REDIRECT_URI`.
- Configure the same URI as a valid post-logout redirect URI in the Keycloak
  client.
- No database migration is introduced by this release.
- Trusted-proxy/oauth2-proxy mode is unchanged.

## Validation

Before using this patch in the demo environment, validate:

- `npm run lint`
- `npm run test`
- `npm run test:e2e`
- `npm run test:security-runtime`
- `npm run build`
- `npm run release:check`
- Login through Keycloak works
- Logout redirects through Keycloak logout
- A later app visit requires a fresh Keycloak login instead of silently
  reusing the previous SSO session
