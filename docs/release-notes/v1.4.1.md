# v1.4.1 - Native OIDC login gate

## Summary

`v1.4.1` tightens native OIDC browser access for demo and future production
deployments. When `AUTH_MODE=native-oidc` and `REQUIRE_AUTH=true`, unauthenticated
browser requests for the React app are redirected to `/auth/login` instead of
receiving the SPA shell.

The release also fixes stale frontend state after logout or session expiry. The
UI now treats `/api/session` as the first source of truth, clears cached domain
data when authentication is missing, disables write actions, and removes the old
user display state.

## Upgrade notes

- No database migration is introduced by this release.
- No Keycloak client change is required when native OIDC already works with
  `v1.4.0`.
- Personal calendar feeds under `/calendar/<token>.ics` remain bearer-token
  based and do not require an interactive browser login.
- Trusted-proxy/oauth2-proxy mode is unchanged and remains available as a
  transition and rollback path.

## Validation

Before using this patch in the demo environment, validate:

- `npm run lint`
- `npm run test`
- `npm run test:e2e`
- `npm run test:security-runtime`
- `npm run build`
- `npm run release:check`
- Unauthenticated `GET /` redirects to `/auth/login`
- Login through Keycloak works
- Logout removes the displayed user and a subsequent browser visit redirects to
  Keycloak again
