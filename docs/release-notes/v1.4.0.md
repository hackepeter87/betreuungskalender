# v1.4.0 - Native OIDC authentication

## Summary

`v1.4.0` introduces native OIDC authentication as the target architecture for
Betreuungskalender. The app handles Authorization Code + PKCE login directly,
uses server-side login state and sessions, maps validated Keycloak claims into
the existing `app_users` model, and keeps tokens out of browser storage.

The trusted-proxy/oauth2-proxy deployment path remains available as a
transition and rollback mode. It is not removed by this release.

## Upgrade notes

- Review [native-oidc-keycloak-podman.md](../native-oidc-keycloak-podman.md)
  for a fresh native OIDC installation.
- Existing oauth2-proxy deployments must follow
  [native-oidc-migration-rollback.md](../native-oidc-migration-rollback.md)
  instead of changing auth mode ad hoc.
- Keep `compose.oidc.yml`, `oauth2-proxy.cfg`, the old Keycloak redirect URI,
  and the pre-migration config backup until native OIDC has been verified live.
- Do not publish or deploy this release until release validation, archive
  checksum verification, runtime verification, and auth-role smoke tests pass.

## Migration impact

Native OIDC uses server-side SQLite tables for login state and opaque session
metadata. Tokens are not stored in the browser and raw token values must not be
logged. Existing `app_users` records continue to provide the stable internal
identity and role surface used by API authorization and audit metadata.

Rollback to trusted-proxy remains supported by restoring the prior `.env`,
`compose.oidc.yml`, and `oauth2-proxy.cfg`, then starting the oauth2-proxy
Compose topology again.

## Security notes

- Production native OIDC should use `OIDC_REQUIRE_ROLE_CLAIM=true`.
- Keycloak must emit the configured groups claim into the ID token.
- Users without a configured role group are rejected in native mode.
- Session cookies are opaque, `HttpOnly`, `SameSite=Lax`, and `Secure` in
  production.
- Logs must not contain authorization codes, access tokens, ID tokens, refresh
  tokens, raw claims, session cookies, nonce values, PKCE verifiers, or client
  secrets.

## Validation

Before publication, validate:

- `npm run lint`
- `npm run test`
- `npm run build`
- `npm run test:security-runtime`
- `npm run test:e2e`
- `npm run release:check`
- Native OIDC browser smoke test for admin, parent, readonly, no-group user,
  logout, and session expiry
- Rollback smoke test back to trusted-proxy in staging
